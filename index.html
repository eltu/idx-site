Why idx

Built for how developers actually search

grep finds strings. idx finds relevant code.

🎯

Relevance ranking

BM25 scoring with a filename-match bonus. Files whose names match your query always surface first.

⚡

Instant results

Pre-built inverted index — no disk scanning at query time. Results arrive before grep opens its first file.

🔒

Offline & private

Everything runs locally. No telemetry, no cloud indexing, no data leaving your machine.

🔬

Precision filters

Narrow by extension (--ext go) or path (--path internal/core). AND / OR with relaxation fallback.

🔄

Always in sync

File saved? Index updated. The background daemon tracks every change — no manual syncs during development.

📖

Read-aware ranking

Files you open frequently get a ranking boost in future searches — the index learns your workflow.

Quick start

Up and running in minutes

Requires a Git repository. No build tools needed.

1

Install via Homebrew

One command. Works on macOS and Linux. No Go required.

2

Index your project

Run once per project. Builds the BM25 index for every directory.

3

Enable background sync

Start once. The daemon keeps the index current with every file save.

4

Search

AND / OR logic, path and extension filters, results ranked by relevance.

zsh
# 1. install $ brew tap eltu/idx && brew install idx   # 2. index your project $ idx init ✓ indexed 847 files in 312ms   # 3. enable background sync $ idx daemon enable . ✓ daemon started (pid 48291)   # 4. search $ idx search "rate limit" --ext go internal/core/services/search/limiter.go 0.97

AI editors

Replace grep in your AI sessions

One command installs the idx skill in your editor. grep and rg calls are intercepted — your AI gets ranked, compact results instead of raw line dumps.

↑speed Faster queries Pre-built index means no disk scanning at query time — speed advantage over grep grows with codebase size.
↓noise Less context consumed Ranked results surface only the relevant files — not every line that matches a pattern. Less output, more signal for your AI.
↓searches Fewer tool calls Relevance ranking gets the AI to the right file faster — fewer back-and-forth iterations per task.
BM25scored Ranked by relevance The most relevant file is always first. Your AI reads less noise to reach the right answer.
